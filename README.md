## Case Study 01: Exploratory Testing - Amazon.com.br

**Objetivo:** Validar a integridade da interface e a experiência do usuário (UX) em fluxos críticos de e-commerce.

### Test Scope (Escopo do Teste)
* **Ambiente:** Production (Web)
* **Foco:** Carrinho de Compras e Detalhes do Produto.
* **Técnica:** Session-based Exploratory Testing.

---

## Bug Reports Identificados

### [BUG-001] – Sobreposição do campo de quantidade (Input Overlap)
- **Severity:** S3 (Medium) | **Priority:** P2 (High)
- **Descrição:** Falha de renderização onde o campo de input sobrepõe rótulos e botões, dificultando a interação no carrinho.
- **Evidência:** [Visualizar imagem](./evidence/bug-001-overlap.png)

### [BUG-002] – Falha de Responsividade (Layout Overflow)
- **Severity:** S3 (Medium) | **Priority:** P2 (High)
- **Descrição:** Em resoluções verticais (1080x1920), o widget lateral do carrinho é cortado, impedindo a visualização do resumo.
- **Evidência:** [Visualizar imagem](./evidence/bug-002-responsividade.png)

### [IMP-001] – Melhoria de Usabilidade (Readability Issue)
- **Type:** UX Improvement | **Priority:** P3 (Normal)
- **Descrição:** Ausência de limites de largura em textos longos em modo retrato, prejudicando a legibilidade das especificações.
- **Evidência:** [Visualizar imagem](./evidence/imp-001-texto.png)
